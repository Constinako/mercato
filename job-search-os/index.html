Set it up

About 15 minutes. No coding, and if you use the Claude desktop app, no terminal either.

1
What you need
Claude Pro or Max subscription (the free tier will not work) Mac or Windows ~15 minutes
2
Download the OS

Download the zip below and unzip it somewhere permanent you'll remember, like your Documents folder. The unzipped folder is called job-search-os. That folder is the operating system.

↓ Download job-search-os.zip

3
Get Claude Code (pick one path)
Easiest: the Claude desktop app Download the Claude app for Mac or Windows from claude.ai/download and sign in. Claude Code is built in: it's the Code section in the app. No terminal, nothing to install beyond the app.
Alternative: the terminal Open Terminal (Mac: ⌘+Space, type “Terminal”) or PowerShell (Windows: Start menu, type “PowerShell”). Paste the install command for your system: Mac curl -fsSL https://claude.ai/install.sh | bash, Windows irm https://claude.ai/install.ps1 | iex. Then claude --version to confirm, and sign in on first run.
4
Open Claude Code in your job-search-os folder

This is the step people miss, so here it is precisely. Claude Code always works “inside” one folder, and all the OS features only exist when that folder is the one it's opened in.

In the desktop app Open the Code section → New session → it asks which folder → choose your unzipped job-search-os folder.
In the terminal Type cd  then drag the job-search-os folder onto the terminal window (it fills in the path), press Enter, then type claude.
5
Type /start

In your first session, type /start and press Enter. The course takes over from there: it introduces itself, checks your setup, and guides you through everything one step at a time. You never need to come back to this page unless something breaks.

Every time you come back: open a new session and select your job-search-os folder again (desktop app: Code → New session → choose the folder · terminal: cd into it, then claude). Not sure where you are? Ask Claude “which folder am I in?” If the OS commands don't appear when you type /, you're in the wrong folder.
If something's off
  • • “command not found: claude” → close and reopen the terminal, or use the desktop app path instead
  • • /start doesn't show up → you're not in the job-search-os folder; redo step 4
  • • login loops or errors → check your Pro/Max subscription is active at claude.ai, then sign in again
